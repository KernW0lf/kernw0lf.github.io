Building JNI from NDK for Android

JNI allows Java Code running in a Java Virtual Machine (JVM) to call and be called by native applications and libraries written in other languages, such as C and C++. Here’s a simple example using Android Studio.

Step 1: Set up your Android Studio

  • Open Android Studio and create a new Android Project
  • In your project, create a new directory called jni at the same level as your src and res directories

Step 2: Write a native method in Java

Create a new Java Class with a native method. This will be the class that contains the JNI interface.

// MyClass.java
public static MyClass {
  static {
    System.loadLibrary("mylibrary"); // Load the native library
  }

  // Native method declaration
  public native String helloFromJNI();
}

Step 3: Generate the header file

Use the following command to generate a header file from your Java Class:

$ javac -h . MyClass.java

This will generate a file named MyClass.h

Step 4: Write the Native Implementation

Create a new C or C++ file, e.g., MyClass.c and implement the natie method.

// MyClass.c
#include "MyClass.h"
#include <string.h>

JNIExport jstring JNICALL Java_MyClass_helloFromJNI(JNIEnv *env, jobject instance){
  return (*env)->NewStringUTF(env,"Hello from JNI");
}

Step 5: Create the Android.mk file

Create a file named Android.mk in the jni directory to specify how the native code should be built.

# Android.mk
LOCAL_PATH  :=  $(call my-dir)

include $(CLEAR_VARS)

LOCAL_MODULE  :=  mylibrary
LOCAL_SRC_FILES :=  MyClass.c

include $(BUILD_SHARED_LIBRARY)

For building on Linux

Make two separate directories libs and jni

$ mkdir libs jni
$ mv Android.mk MyClass.c MyClass.class MyClass.h MyClass.h jni
$ cd jni
$ /path/to/ndk-build
$ ls ../libs/
arm64-v8a armeabi-v7a x86 x86_64

If you get “Android NDK: APP_PLATFORM not set” error, then Create and Open Application.mk file in the jni directory of your Android Project. Add the following line explicitly to it:

APP_PLATFORM  :=  android-26

Step 6: Build the native library

Open a terminal and navigate to your project’s root directory. Run the following commands to build the native library:

$ cd app/src/main
$ /path/to/ndk-build

Step 7: Link the native library in your app

In your build.gradle file, add the following lines to link the native library:

android {
  // ...
  defaultConfig {
    // ...
    externalNativeBuild {
      ndkBuild {
        path 'src/main/jni/Android.mk'
      }
    }
  }
  // ...
}

Step 8: Use the native method in Java

Now you can use the native method in your Java code:

// MainActivity.java

  @Override
  
}
